# M9 — Nomination Status & Payment

## Overview
Handles everything that happens after a nomination is submitted: real-time status tracking from STS Platform, viewing EPB and schedule details, uploading Proof of Payment, and receiving payment verification results. This module ends when status is `PAYMENT_CONFIRMED` and the voyage is authorized to start.

## Boundaries
**In scope:**
- Receiving and displaying status updates from STS Platform (Pending, Approved, Need Revision)
- Viewing EPB (Estimasi Perkiraan Biaya) and schedule details (anchor point, ETB)
- Revising nomination on Need Revision and re-submitting to STS
- Uploading Proof of Payment (PDF/JPG/PNG, max 5MB)
- Sending payment proof to STS Platform and receiving verification result (Confirmed/Rejected)
- Customer notifications on every status change

**Out of scope:**
- Initial nomination submission (M8)
- Dashboard summary view (M10)
- Actual payment processing or billing (STS Platform)
- Nomination approval logic (STS Platform)

## Dependencies
| Dependency | Reason |
|-----------|--------|
| M8 Nomination Request Submission | A nomination must exist in PENDING or later status |
| M7 Customer Authentication | Customer must be authenticated |
| STS Platform API (outbound) | Send payment proof; receive webhook for status updates |
| STS Platform Webhook (inbound) | LPS receives status changes: Approved, Need Revision, Payment Confirmed/Rejected, EPB data |

## Downstream
| Module | Dependency |
|--------|-----------|
| M10 Customer Dashboard & Monitoring | Displays active nominations and voyage start trigger |

## Key Decisions
- LPS does NOT approve nominations or verify payments — all decisions come from STS Platform via webhook/callback.
- EPB is generated by STS Platform and received by LPS; LPS displays it read-only.
- Payment proof is stored in LPS and forwarded to STS — LPS does not make payment decisions.
- STS webhook must be authenticated (HMAC signature or API key header).
- If STS webhook is unavailable, LPS must support a polling fallback for status updates.
