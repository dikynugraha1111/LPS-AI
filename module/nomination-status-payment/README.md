# M9 — Nomination Status & EPB Confirmation

## Overview
Handles everything that happens after a nomination is submitted up to the first payment proof upload (EPB Confirmation). This module ends when the customer submits their first proof of payment — after which data moves to the EPB & Invoice menu (M9b) for the full payment verification cycle.

## Boundaries
**In scope:**
- Receiving and displaying status updates from STS Platform (Pending, Approved, Need Revision)
- Viewing EPB (Estimasi Perkiraan Biaya) and schedule details (anchor point, ETB) — read-only
- Revising nomination on Need Revision and re-submitting to STS
- EPB Confirmation: customer uploads first Proof of Payment and submits
- After Submit on EPB Confirmation: data is moved/handed off to M9b (EPB & Invoice menu)
- Customer notifications on every status change

**Out of scope:**
- Initial nomination submission (M8)
- Payment verification cycle (Pending Review → Payment Reject → Paid) — handled by M9b
- Dashboard summary view (M10)
- Actual payment processing, billing calculation, or invoice generation (STS Platform)
- Nomination approval logic (STS Platform)

## Dependencies
| Dependency | Reason |
|-----------|--------|
| M8 Nomination Request Submission | A nomination must exist in SUBMITTED/PENDING or later status |
| M7 Customer Authentication | Customer must be authenticated |
| STS Platform Webhook (inbound) | LPS receives status changes: Approved, Need Revision, EPB data |
| M9b EPB & Invoice | Receives the payment proof data after customer submits EPB Confirmation |

## Downstream
| Module | Dependency |
|--------|-----------|
| M9b EPB & Invoice | Receives handoff after EPB Confirmation submit |
| M10 Customer Dashboard & Monitoring | Displays active nomination statuses |

## Key Decisions
- M9 scope is **strictly up to the first payment proof submission**. The full payment verification cycle (Unpaid → Pending Review → Payment Reject / Paid) belongs to M9b.
- After customer Submits on EPB Confirmation page, data is moved to M9b. M9 does NOT track WAITING_PAYMENT_VERIFICATION, PAYMENT_CONFIRMED, or PAYMENT_REJECTED statuses.
- EPB is generated by STS Platform and received by LPS; LPS displays it read-only.
- STS webhook must be authenticated (HMAC signature or API key header).
- If STS webhook is unavailable, LPS must support a polling fallback for status updates.
