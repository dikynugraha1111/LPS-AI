# M8 — Nomination Request Submission

## Overview
Handles the complete nomination creation flow: filling the nomination form, uploading supporting documents, saving as draft, and final submission. On submit, a Nomination Number is auto-generated and the data is forwarded to STS Platform via API. This module ends when the nomination is successfully submitted; subsequent tracking is handled by M9.

## Boundaries
**In scope:**
- Nomination form creation and editing (including Draft state)
- Supporting document uploads
- Auto-generation of Nomination Number on submit
- Sending nomination payload to STS Platform API

**Out of scope:**
- Status tracking after submission (M9)
- Payment flow (M9)
- Dashboard display of nomination summaries (M10)

## Dependencies
| Dependency | Reason |
|-----------|--------|
| M7 Customer Authentication | Customer must be authenticated (role = customer) to create nominations |
| STS Platform API | Nomination data is sent to STS on submit |

## Downstream
| Module | Dependency |
|--------|-----------|
| M9 Nomination Status & Payment | Operates on nominations created by M8 |
| M10 Customer Dashboard & Monitoring | Displays nomination summaries from M8/M9 |

## Key Decisions
- Draft nominations are stored locally in LPS; they are NOT sent to STS Platform until the customer explicitly submits.
- A Nomination Number is generated by LPS on submit (not by STS Platform).
- Document uploads are stored in LPS file storage; document metadata is included in the STS API payload.
- If STS API call fails after submit, the nomination is marked `SUBMIT_FAILED` and the customer is notified to retry.
